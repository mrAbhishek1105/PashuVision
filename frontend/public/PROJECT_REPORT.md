## 1. Abstract
The "**PashuVision System**" is an AI-powered prototype designed to assist farmers, veterinarians, and cattle enthusiasts in correctly identifying Indian cattle and buffalo breeds. Utilizing Deep Learning (Convolutional Neural Networks) and Transfer Learning (MobileNetV2), the system analyzes user-uploaded images to predict the breed with high accuracy. This report details the development lifecycle, including data collection, model training, backend API development, and the creation of a responsive user interface.

## 2. Introduction
### 2.1 Background
India possesses a vast biodiversity of livestock, with over 50 registered cattle breeds and 19 buffalo breeds. However, cross-breeding and lack of formal records make identification difficult for the uninitiated. Correct identification is crucial for:
- Breeding programs (preserving genetic purity).
- Veterinary care (breed-specific diseases).
- Economic valuation (milk yield estimations).

### 2.2 Objective
To build a web-based full-stack application that:
1.  Accepts an image of a cow or buffalo.
2.  Preprocesses the image for AI analysis.
3.  Classifies it into one of 5 target breeds.
4.  Returns detailed information (habitat, diet, milk yield).

## 3. System Architecture
The project follows a standard 3-tier architecture:

1.  **Presentation Layer (Frontend)**: 
    -   Built with **React + Vite**.
    -   Uses **React Router** for navigation.

2.  **Application Layer (Backend)**:

3.  **Data & Intelligence Layer (ML Engine)**:
    -   **MobileNetV2** (pre-trained on ImageNet) fine-tuned for livestock.
    -   **TensorFlow/Keras** framework.
    -   JSON-based data store for breed attributes.

## 4. Methodology
### 4.1 Data Collection
A custom dataset structure was created to classify 5 major breeds:
-   **Cattle**: Gir, Sahiwal, Red Sindhi.
-   **Buffalo**: Murrah, Jaffarabadi.

### 4.2 Data Preprocessing
-   **Resizing**: All images resized to 224x224 pixels.
-   **Normalization**: Pixel values scaled to `[-1, 1]` to match MobileNetV2 requirements.
-   **Augmentation**: To prevent overfitting, the following transformations were applied during training:
    -   Rotation (±20°)
    -   Width/Height Shift (20%)
    -   Shear & Zoom (20%)
    -   Horizontal Flip

### 4.3 Model Architecture
We utilized **Transfer Learning** with MobileNetV2.
-   **Base Model**: MobileNetV2 (weights='imagenet', include_top=False).
-   **Custom Head**:
    -   GlobalAveragePooling2D
    -   Dense Layer (128 units, ReLU activation)
    -   Dropout (0.5) for regularization
    -   Output Layer (Softmax activation for 5 classes)

### 4.4 Training Strategy
1.  **Phase 1 (Feature Extraction)**: Frozen base model, trained only the custom head for 10 epochs.
2.  **Phase 2 (Fine-Tuning)**: Unfrozen the top 50 layers of MobileNetV2, re-trained with a very low learning rate (`1e-5`) to adapt high-level features to livestock textures.

## 5. Implementation Details
### 5.1 Backend (Flask)
The core logic resides in `app.py`.
-   `load_resources()`: Safely loads the `.h5` model and `class_map.json`.
-   `prepare_image()`: Ensures input image matches training transformations.
-   `predit()`: The main endpoint returns JSON containing `breed`, `confidence`, and `info`.

### 5.2 Frontend (React)
The UI is broken down into:
